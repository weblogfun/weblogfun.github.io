[版本更新] 快速回顧 微軟發布 KB5010386 (Windows 11 作業系統組建 22000.493)

發表於 2022-02-13 分類於 windows
文章字數： 1.5k 所需閱讀時間 ≈ 3 分鐘

前言

隨著不少新的電腦已搭載 Windows 11 出貨，我相信目前的 Windows 11 已經有一定的穩定度，而我此次更新到 Windows 11 後，會持續觀注系統補丁，而且不一定每一個補丁釋出，都立馬跟著更新，主要是希望，這一回升級到 Windows 11 至少要能夠穩穩用個 1 年。

今天升級的機器，已經經歷過一次 Windows 11 升級，升級後掛點前，機器上的韌體更新了幾次，考量到微軟也陸續提供系統補丁，就在前天，注意到 Windows 11 推動的升級訊息時，我就前面幾點的考量後，便決定再次將系統升級到 Windows 11(花費大概 1 小時 15 分鐘)。恰巧的是我完成 Windows 11 升級的當天，正好碰上微軟釋出 2022 年 2 月 8 日 —KB5010386 (OS Build 22000.493) 這支補丁，並且，直到今天才收到更新的通知。

說真的，這樣也好，通常更新跑第一，都需要一點冒險精神，這次升級到 Windows 11，我求的是穩定度，不會想要賭一把。

更新前內容概覽

我先讀過 2022 年 2 月 8 日 —KB5010386 (OS Build 22000.493) 裡的說明。

我特別注意到 2 個部分： 其一，中文輸入法的可靠性(Reliability)：可能是我中字打字還不夠多，也可能是我在 Windows 11 的機器上，搭配使用的 App 也不夠多元與廣泛，我自己並沒有發現任何中文輸入法不夠可靠的問題，另外，我使用的是嘸蝦米輸入法是否包涵在這次的更新範圍(…這麼說好了，我個人不太能夠從微軟的說明裡，具體地瞭解到更新「中文輸入法可靠性」指的是什麼)。

其二，Your Microsoft Account 與 HelpWith：這 2 個地方，應屬介面上的更新，HelpWith 會提供一些說明連結供使用者在需要時點傳送門上網瀏覽。

更新之前，我先跑一趟 Microsoft Account 與【系統〉顯示器】，去看原本的模樣。

Microsoft Account 的部分，我總覺得更新之前，就跟影片介面的樣子，沒有太大差別。這個部分，可以就下面 2 張圖做比較。

看一下更新的時間後，也讓另外一支補丁，一併安裝進來。

更新很快就完成，我特別先跑一趟 Microsoft Account 與【系統〉顯示器】，想看看更新之後長什麼樣？

答案是「都沒有變」(…呃，詳見下圖)。我在更新完幾個小時後，還重開機確認是沒有，這個部分，Microsoft Account 的部分，好像更新就長這樣，而 HelpWith 的部分，可能要另外開啟，或是中文版的部分，還不在這次的更新補丁裡…吧…？！

剛更新完時，我特別先再去看一下，系統是不是順利將補丁更新上來。

後記

關於 KB5010386 補丁的更新內容，也可參考 微软 Win11 Build 22000.493 (KB5010386) 发布：修复 SSD 硬盘性能 Bug，设置新增管理订阅，但我這次沒有特別針對 SSD 讀取速度去做測試，不確定這支補丁是不是真的讓 SSD 的讀取速度改善，就我目前的使用上，感覺也並不是那麼特別明顯。

另外，服務堆疊更新 (SSU) 是什麼？有興趣的伙伴可以自行上網搜尋，或是參考 服務堆疊更新 (SSU)：常見問題集。
