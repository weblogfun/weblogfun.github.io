WinLibs：Windows 上免費、好設定的 Compiler

發表於 2024-04-01 分類於 windows
文章字數： 1.1k 所需閱讀時間 ≈ 2 分鐘

前言

  1. 之前，在撰寫安裝過程紀錄時，我曾說。

    我在整理自己的紀錄時，發現「安裝 MSYS2 與 MinGW」是相對比較麻煩的，未來如有參考需要時，我可能會先從這一個部分開始安裝與設定。

  2. 今天要紀錄的 WinLibs，算是另外一個在 Windows 上設定 Compiler 時，很不錯的選擇。

下載與設定 WinLibs

  1. 下載 WinLibs 上的編譯器時，可能會想，是要下載 MSVCRT？還是 UCRT？關於這一點，官網有說明，官網也直接給結論：以 UCRT 為主。

    Unless you are targetting older versions of Windows, UCRT as runtime library is the better choice, as it was written to better support recent Windows versions as well as provide better standards conformance.

  2. 無論是選擇 7-Zip 或是 Zip 壓縮檔，解壓的路徑，我自己的習慣是直接丟到系統硬碟的根目錄底下(也就是C:)。

  3. 接著，要設定 Windows 的 PATH 環境變數。要快速地進入「系統內容」，可以開 Windows 的 Terminal (CMD) 或是按住「Win+R」，執行 systempropertiesadvanced。

  4. 點「環境變數」。

  5. 從「系統變數」欄位中，找到「Path」，點選後，再點「編輯」。

  6. 把「C:\mingw64\bin」這個路徑，新增進去吧！

  7. 如果有已開啟的 Windows Termial (CMD) 的話，關掉並重新啟動 Terminal，如果沒有，直接啟動一個新的，執行下列命令，確認安裝完成。

  8. 大致上，就可以拿個 Code 來測試有沒有成功。

後記

  1. 本次紀錄的系統環境是 Windows 10 家用版 22H2，OS 組建 19045.4170，體驗 Windows Feature Experience Pack 1000.19054.1000.0。

  2. 日前，我分成 3 個 EP 紀錄一下安裝 Visual Studio Code (又稱 VSCode)，設定 C++ 的程式設計環境，而這 3 個 EP 如下。
    EP 1：安裝 Visual Studio Code 主程式
    EP 2：安裝 MSYS2 與 MinGW
    EP 3：設定程式設計專案路徑，安裝 Visual Studio Code 語言套件與 C++ 相關套件
