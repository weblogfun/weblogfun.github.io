在 Acer Aspire X3600 上面安裝 ubuntu 8.04

發表於 2008-05-19 分類於 linux
文章字數： 1.5k 所需閱讀時間 ≈ 3 分鐘

初極狹，纔通人；復行數十步，豁然開朗。 (( 出自桃花源記（陶淵明），相關訊息可以參閱：桃花源 - Wikipedia 與陶淵明 - Wikipedia。 ))是本次我試著讓 ubuntu 8.04 從舒適卻受限的 VMWare 環境往實機安裝的最佳心境寫照，也了解到一直以來 ubuntu 能夠以 700 MB 的 ISO 檔供使用者下載的原因，再回頭看自己與 ubuntu 共處的經驗，更加確信通往烏托邦的路上，乙太網路線在其中扮演著相當重要的角色。

今次我是 ubuntu 8.04 與 X-BLUE 的媒人，然而更重要的是那條看得見的紅線，絕不能少。談完感受，接下來就分享敝人在下實際撮合雙方的過程…

目前直接使用 ubuntu 8.04 安裝並與 Windows Vista 共存的雙系統設置中，開機的部分會不會像之前在 ubuntu 7.10 那樣不好設定？這個問題，使用 Google 爬文的結果是好像還沒有相關的文章與經驗分享，絕大多數仍舊是在談 ubuntu 7.10，為此不想多做猜想，也不希望因為下不了判斷而盲目測試，更何況早先使用 wubi 的 Install inside Windows 的方法有效，所以我在從雙系統這個方向走時，很快地就決定延用此法練就，經驗值累積後，這乙次我可是乖乖地選擇 English 為預設的安裝語系，相同的部分是將日前備下來的 sources.list 丟到 /etc/apt 路徑底下，再執行 apt-get update 的指令 (( 附帶一提的是 wubi 在 Windows XP 執行 Install inside Windows 相當順利，卻在 Windows Vista 安裝結束時掛點，之前我有同樣的遭遇，但是開機後仍舊可以順利進入烏托邦，今兒個卻不行，只得移除再重新安裝乙次，耐心的等候結果是 wubi 依然沒有回應，所幸能夠進入系統，由於沒深究原因，如果有伙伴們有同樣的遭遇，或許多點等候也能夠解決，但這種非除錯的方法不見得每每奏效。根據官方的釋出說明，最好的方法是把 wubi 拷貝至電腦桌面並執行之，詳可閱 Ubuntu 8.04 Release Notes | Ubuntu 中的 wubi 部分。 )) 。

為何會提到桃花源記，原因就是出在這兒，之前安裝時沒有被載入的 Broadcom 驅動程式，今次我更為謹慎，在 Broadcom 43xx 閱讀 Devices 的頁面資訊，可是安裝後情境依舊，只好硬著頭皮裝上乙太網路線，令人興奮的是系統很快地就回應有新的第三方驅動程式釋出，一步接著一步很就可以通往烏托邦。

還記得 Fedora 9 提到 GNOME 已經附上 Cheese 這支視訊套件嗎？ubuntu 8.04 並沒有在安裝後隨附，卻可以輕鬆地透過 apt-get 事後裝上，我的 Webcam 輕易地被驅動程式支援，呈現出來的結果並不如人意，有點可惜。我找不到圖形裡面去做細部的調整，其他可能的方法，未來再試試。

在清閒的補休假晨間，費時不到 1.5 小時，就可以成功地在 X-BLUE 建立烏托邦，興奮之情難以言喻。擺脫對魔獸世界的掛念後，未來我將把 95% 的工作時間擺在烏托邦，至於剩餘的 5% 還是得靠維斯塔帝國的援助，因為來自惠普世界的 C4385 現階段只與 Windows 以及 Mac OS X 打交道。

ubuntu 8.04 與 X-BLUE 可以 Marriage，我是樂見其成的媒人，也是最大的受益者。
