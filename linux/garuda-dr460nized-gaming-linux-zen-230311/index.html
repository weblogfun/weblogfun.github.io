來玩 Garuda Dr460nized Gaming Linux Zen 230311

發表於 2023-03-22 分類於 linux
文章字數： 1.5k 所需閱讀時間 ≈ 3 分鐘

前言

Garuda Linux 是 Arch Linux 的衍生版本，去年，我第一次接觸這一支 Distro，應該是 DistroWatch 「網頁參觀排名」前 10 名當中，我感到最陌生的一支。

而今，這支 Distro，目前算是我心目中非常喜翻的前 10 名，…正好跟 DistroWatch 上的排名一致。

安裝過程

去年，我的心得是安裝 Garuda 跟 LMDE 5、EndeavourOS 一樣，不把 ISO 下載時間算進來的話，光就整個安裝過程來算，大約 15 分鐘就搞定。

今年，同樣的硬體(本次運行的機器上的 CPU 品牌與型號，一樣是 Intel(R) Core(TM) i5-6400)，整個安裝時間，卻意外地拉長了約 2 倍。

會有這樣的結果，我並不意外。畢竟，我的機器也真的算很老舊，光是還在用硬碟在跑系統，就能夠預測，過了今年，應該跑很多 Linux，都會相當吃力。

同時，也讓我想到前不久，我在寫 Linux Lite 6.2 的 3 天體驗文章時，對 Linux Lite 6.2 的效能感到不是很滿意，而今相較之下，反而認為可能有點太早給 Linux Lite 6.2 下定論，不過，我還是得說，2023 年到目前而言，MX Linux、Garuda 與 Linux Lite 三者之間，MX Linux 側是略勝一籌。

快速回顧與感想

我想，不跟別的 Distro 相比，就跟自己比較的話，以桌面系統使用的感受而言，最明顯的就是在這一個版本裡，有些操作介面與使用邏輯上，我個人是認為有進步的。

Setup Assistant on Garuda Dr460nized Gaming Linux Zen 220329

比方說，Garuda Setup Assistant 去年要透過「Next」(俗稱的「下一步」)才能夠秀出每一種設定或 App 分類。(如下影片)

Garuda Setup Assistant on Garuda Dr460nized Gaming Linux Zen 230319

今年，變成直接以「頁籤」的方式，把所有分類秀出來，，讓使用者從這些「頁籤」找到想要做的設定或是想要安裝的 App 後，再進行後續的執行與安裝的動作。(如下影片，與重點截圖)

除此之外，幾個 Garuda 專屬 GUI 都在使用邏輯上有持平或更好的安排。(下圖)

Garuda Gamer 裡面的 Anbox，安裝之後，無法順利啟動，但，我上次是在實體機器上的 Garuda 運行的結果，而這次是在 VirtualBox 裡的 Garuda，再去試著安裝與執行 Anbox，所以，有機會還是應該要找一部實體機器來玩玩比較準。(下圖)

嘸蝦米輸入法的安裝與設定

  1. Arch Linux 的 Fcitx5 相關套件非常完整，身為 Arch Linux 衍生版本的 Garuda，還可以直接使用 GUI 介面，把常用的套件安裝進來。
  2. 安裝之後，「中州韻」是預設的輸入法。
  3. 嘸蝦米輸入法的部分，要另外再下 sudo pacman -S fcitx5-table-extra 這個指令把表格檔安裝進來，再從 GUI 介面進行新增的動作，要是 Garuda 可以讓這個動作也全部在 GUI 就達成，應該就是更完美的 GUI 化(目前我的經驗，好像只有 openSUSE 做得到)。
