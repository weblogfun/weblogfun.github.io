從 Linux Mint 22 Beta 版升級到正式版？

發表於 2024-07-27 分類於 linux
文章字數： 1.5k 所需閱讀時間 ≈ 3 分鐘

前言

本來，我猜 Linux Mint 22 Beta 版之後，可能要等到 8 月初才會迎來官方將正式版釋出。沒有想到，我在寫 Beta 版的使用心得時，Linux Mint 團隊，也在同一天釋出正式版。就這樣一前一後，是有那麼一點巧。也讓我想說，要不要趁這個機會，多瞭解一下 Beta 版與正式版之間的差異，以及嘗試與評估一下直接從 Beta 版升級到正式版的可能性。

爬文參考 vs. 實際觀察

爬文參考（用 Google 爬到 Linux Mint 論壇上的討論串）

  1. 首先，同時也是最主要的參考是這一篇討論：Upgrade Linux Mint 21.3 to 22 - Linux Mint Forums。

  2. 從討論來看，發問的伙伴，想要知道能不能從 Linux Mint 21.3 升級到 Linux Mint 22 Beta，最後再升級到 Linux Mint 22 正式版。

  3. 再從伙伴們的回應來看，似乎可以，又似乎不行。以我自己的經驗去推測的話，我比較相信無法從正式版轉成 Beta 版。但，我也不確定能不能直接從 Beta 版，升級到後續的正式版。

運用 VirtualBox 實際觀察

  1. 從 LIVE CD 出發做觀察：這沒有什麼技巧，純綷就是分別從 Beta 版 ISO 檔（linuxmint-22-cinnamon-64bit-beta.iso）與正式版 ISO 檔（linuxmint-22-cinnamon-64bit.iso）開機後，觀察彼此之間的不同。

  2. 進入 LIVE CD 的系統後，我觀察一下，發現開啟 Beta 版裡的「系統資訊」（System Info）是看不到 Beta 字樣的。

  3. 正式版已經將安裝程式，提高語言支持度，所以，不像 Beta 版會是一堆亂碼。

試試從 Beta 版升級

  1. 先將 mintupdate 升級到 6.1.5 版。

  2. 升級前，我會先把軟體倉庫「切換到本地的鏡像網站」。

  3. 「切換到本地的鏡像網站」時，記得要[更新 APT 快取]。

  4. 開始更新 mintupdate。

  5. 接著，就可以更新最新的 Linux Mint 22 核心與相關套件。

  6. 重新啟動系統後，下一道指令 uname -r 看一下新版本的 Kernel 是否順利安裝與載入。

  7. 也可以從「系統資訊」（System Info）看看其他系統相關訊息。

結論

  1. 因為爬到 Upgrade Linux Mint 21.3 to 22 - Linux Mint Forums 這一串討論，讓我萌生一些實驗的想法。
  2. 本篇所做的比較與實驗是比較粗糙的，僅管如此，我自己走完後，會認為從 Beta 版升級到正式版的說法，應該是比較可信的。
  3. 對於已經在 Beta 版做了不少事，不想重灌的伙伴來說，只要繼續升級就可以，這樣算是滿方便的。如果還是想從乾淨的系統起步，重新安裝也不會花太多時間，只是如果要把 Beta 版的設定與檔案轉移到正式版，砍掉重練的重新安裝，要花多少時間，就因人而異。我相信，對於熟悉轉移的伙伴而言，可能已經熟練，也不一定會在乎花費的時間多寡。
