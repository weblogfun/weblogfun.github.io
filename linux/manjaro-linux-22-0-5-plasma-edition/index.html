來玩 Manjaro Linux 22.0.5 Plasma Edition

發表於 2023-04-18 分類於 linux
文章字數： 1.6k 所需閱讀時間 ≈ 3 分鐘

前言

維基百科的介紹上是這麼寫著的(如下)

Manjaro Linux（或簡稱 Manjaro）是基於 Arch Linux 的 Linux 發行版，使用 Xfce 、GNOME和 KDE Plasma 作為預設桌面環境，和 Arch 一樣，採用滾動更新。其目標是為 PC 提供易於使用的自由的作業系統。

當我正在撰寫這一篇文章時，Manjaro 在 DistroWatch.com 上的排名是第 4 名。說真的，從我開始重新開始寫部落格以來，Manjaro 的排名一直都在前 5 或前 10，我相信 Manjaro 確實有自己獨有的魅力與穩定忠實的支持者。

我相信會選擇 Manjaro 的伙伴，多半是衝著桌面環境而來，對我來說，我很喜歡 Manjaro 的桌面環境，尤其是那獨樹一格的桌面顏色配色。

快速回顧與感想

  1. 這次，我的選擇是 Plasma Desktop：我之前的經驗都是選用 XFCE Desktop，但，這一次下載 ISO 的時候，我發現 Plasma Desktop 在下載頁的左上角，也就是排位上的…第一位(下圖)。

    想不到，我在寫這篇文章時，也注意到 DistroWatch.com 有一篇 Review: Manjaro Linux 22.0, Trisquel GNU/Linux 11.0(截圖連結另開)。作者 Jesse Smith 也跟我做出同樣的選擇，但可能是安裝的環境不同，Jesse Smith 的遭遇就沒有那麼幸運，他說(如下)

    About halfway through the install process Calamares crashed and left my hard drive in an unbootable state. At this point I’d booted Manjaro media four times and had no success in fetching packages or getting the operating system installed. I decided to try once more, downloading the Xfce Minimal edition which is 3.0GB in size.

    由於，我沒有再去爬文瞭解其他使用者是不是有同樣的狀況，因此，Jesse Smith 的情況，可能是個案，只是，我也得說，我多半是在 VirtualBox 上完成安裝，VirtualBox 的環境相對是單純許多的，同時，也比較不容易發生安裝過程就 Crash 的狀況。

  2. 從這一次的使用去看看 KDE：當年，KDE 與 Gnome 兩大桌面環境的時代裡，我對 KDE 的第一印象是好的，但，實際安裝使用後，效能、穩定性都不如預期，直到在 openSUSE 上的使用經驗，才慢慢翻轉這樣的想法，這真的算是我的個人使用經驗與偏見。而今，我在各個不同的 Distro 上，都有機會碰到 KDE，即使 KDE 不一定是我的首選，我也不會再認為 KDE 有什麼可以嫌棄的地方。

  3. 初次啟動系統後的桌面環境概覽：忘了是那一版的 Manjaro Linux 開始，介面上將圓角導入，可能已是近年的流行趨勢，總覺得看起來挺順眼的，並無違和感。

有關中文輸入法的安裝與建議

  1. 輸入法框架間的選擇：有關這一點，在這個時間點上，我自己會以 iBus 或 fcitx 之間去做出選擇為主。
  2. fcitx 會是首選：而且，在 fcitx(4) 或 Fcitx5 之間，我會建議直接就選用 Fcitx5。
